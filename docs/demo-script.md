## ก่อน Demo ทุกครั้ง (Pre-flight checklist)

- [ ] เปิด Supabase project เองล่วงหน้า 5-10 นาที (กัน pause เพราะไม่มีคนใช้ 7 วัน)

- [ ] เตรียมคำถามสำหรับ chatbot ไว้ล่วงหน้า 3-5 ข้อ ที่รู้ผลลัพธ์คร่าวๆ
- [ ] เปิดวิดีโอ demo สำรองไว้ในเครื่อง (เผื่อ live ใช้ไม่ได้ตอนหน้างาน)
- [ ] เช็ค wifi/เน็ตที่ห้อง present ล่วงหน้าถ้าเป็นไปได้

## ลำดับการ Demo (ประมาณ 8-10 นาที)

### 1. เปิดด้วย Home (30 วิ)
พูดสั้นๆ ว่า DME Explorer คืออะไร แก้ปัญหาอะไร (ข้อมูลกระจัดกระจาย นักเรียนม.ปลายและนักศึกษาปีหนึ่งไม่มีที่รวมข้อมูล) โชว์หน้า Home ที่มี summary + call-to-action

### 2. Curriculum Roadmap (1.5 นาที)
- คลิกเข้าดู roadmap 4 ปี
- คลิกเปิดวิชาหนึ่งวิชาโชว์ description + prerequisite
- เน้นว่าข้อมูลหลักสูตรนี้เป็นของจริงจาก DME KKU (ไม่ได้ทำ mock) แบ่งเป็น 4 สาย **AI / Digital Media / Interactive / Software** ตรงกับ major elective จริง

### 3. Career Explorer (1.5 นาที) ⚠️ จุดเสี่ยง (พึ่ง JSearch API ภายนอก)
- เลือก filter สนใจ (เช่น Game Dev) โชว์ตำแหน่งงานที่เปิดรับจริงตอนนี้
- ถ้า API ล่มสด: มี fallback data cache ไว้ล่วงหน้า สลับมาพูดว่า "นี่คือตัวอย่างข้อมูลที่ระบบดึงมาปกติ" แทนการหยุดชะงัก

### 4. Tuition & Fees (1.5 นาที) — หน้าที่เพิ่มใหม่
- เลือก student type (Thai / Mekong Region / International) โชว์ตัวเลขค่าเทอมเปลี่ยนทันที
- สลับ toggle Per Semester ↔ Full 4 Years โชว์การคำนวณ real-time
- เปิด Fee Breakdown โชว์รายละเอียด (tuition, one-time fee, living cost โดยประมาณ)
- พูดเน้นจุดขาย: ตัวเลขนี้อ้างอิงจากข้อมูลจริงของโครงการ Studio 4 ก่อนหน้า ไม่ใช่เดาขึ้นมาเอง แต่ **ควรกำชับกับอาจารย์/ผู้ฟังว่าตัวเลขต้องยืนยันกับคณะอีกครั้งก่อนเผยแพร่จริง** (ไม่ใช่ข้อมูลทางการ 100%)

### 5. Student Projects (1 นาที)
โชว์ gallery ผลงานนักศึกษา คลิกดูรายละเอียดหนึ่งชิ้น

### 6. 3D World — CDLC Simulation (2 นาที) ⚠️⚠️ จุดเสี่ยงสูงสุด (WebGL + build ของเพื่อน)
- เข้าหน้า 3D World พูดนำก่อนว่ากำลังจะโหลด (กันดูเหมือนค้าง)
- สาธิตควบคุมตามที่ Thanasoonton กำหนดไว้ (ใส่ controls scheme จริงตรงนี้เมื่อรู้แล้ว เช่น "ใช้ WASD เดิน, ลากเมาส์หมุนมุมมอง")
- เดินสำรวจสั้นๆ ให้เห็นบรรยากาศห้อง CDLC จริง
- **ถ้าเกิดปัญหาโหลดช้า/ค้างหน้างาน**: สลับไปเปิดวิดีโอสำรองที่อัดไว้ทันที พูดว่า "ระบบสามารถรันได้บนเครื่องที่มี GPU เพียงพอ ขอโชว์วิดีโอที่บันทึกไว้แทนเพื่อความรวดเร็ว"

### 7. AI Chatbot (1.5 นาที)
- ถามคำถามที่เตรียมไว้ล่วงหน้า 2-3 ข้อ (เช่น "DME เรียนอะไรบ้าง", "จบแล้วทำงานอะไรได้")
- เน้นว่าใช้ Claude Haiku 4.5 เพราะตอบเร็ว ต้นทุนต่ำ เหมาะกับ FAQ-style

### 8. About / Contact / FAQ (30 วิ)
ผ่านเร็วๆ โชว์ว่ามีครบตามที่ระบุใน scope

### 9. ปิดท้าย (30 วิ)
สรุปจุดต่างจาก competitor ตาม Related Work table (DME-specific + live job data + AI chatbot + interactivity สูง) และพูดถึง future work (multilingual, ขยายไปสาขาอื่น)
